# WMS仓储系统 - 享元模式 + 装饰器模式 实验报告

## 一、项目概述

本项目实现了一个仓储WMS系统的核心功能，结合享元模式和装饰器模式完成货架管理、拣货策略和上架策略的设计。项目使用Java 8 + Maven + JUnit 5。

---

## 二、类图

> 见文件：`docs/class-diagram.puml`

![类图](class-diagram.puml)

### 类图说明

项目分为三个模块：

1. **享元模式模块**（`com.mashang.flyweight`）
   - `ShelfTemplate`：抽象享元接口，定义展示方法（接收外部状态）
   - `ConcreteShelfTemplate`：具体享元类，封装内部状态（类型、长宽高、承重、层数）
   - `ShelfTemplateFactory`：享元工厂，维护模板池，支持禁用功能
   - `Shelf`（`com.mashang.model`）：持有模板引用 + 外部状态（编号、库区、占用状态）

2. **拣货策略装饰器**（`com.mashang.decorator.picking`）
   - `PickingStrategy`：抽象构件接口
   - `NormalPicking`：具体构件（按库位顺序拣货）
   - `PickingDecorator`：抽象装饰器（含优先级）
   - `WavePickingDecorator`、`ColdChainPickingDecorator`、`FragilePickingDecorator`：具体装饰器

3. **上架策略装饰器**（`com.mashang.decorator.putaway`）
   - `PutawayStrategy`：抽象构件接口
   - `NormalPutaway`：具体构件（空闲库位就近上架）
   - `PutawayDecorator`：抽象装饰器（含优先级）
   - `BatchControlDecorator`、`ExpiryPriorityDecorator`、`WholeSplitDecorator`：具体装饰器

### 4.2 关键代码说明

#### 享元工厂 - 模板复用与禁用

`ShelfTemplateFactory` 在构造时预注册4种模板（横梁式、驶入式、穿梭式、轻型置物架），通过 `Map<String, ShelfTemplate>` 池化复用。`getTemplate()` 方法从池中取模板，同一类型返回同一实例。`disableTemplate()` 将类型加入禁用集合，获取时抛出 `IllegalStateException`。

#### 装饰器 - 优先级机制

每个装饰器持有 `priority` 字段（数值越小优先级越高），默认优先级：
- 易碎品拣货：5
- 冷链拣货：10
- 效期优先上架：5
- 批次管控上架：10
- 波次拣货：20
- 整散分离上架：15

#### 整散分离装饰器

`WholeSplitDecorator` 接收 `ShelfTemplate` 参数，根据模板的 `layers` 属性计算整托/散件分层：散件放第1到 `layers/2` 层，整托放 `layers/2+1` 到 `layers` 层。
